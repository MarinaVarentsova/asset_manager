Нужно доработать дизайн загрузчика реестра сертификатов.

Важно:
- Не менять архитектуру.
- Не добавлять базу данных.
- Не хранить историю загрузок.
- Все показатели считать только в момент текущей загрузки Excel.
- Supabase Storage оставить как есть.
- Основной функционал загрузки, проверки, скачивания файла ошибок и генерации JS-файла не ломать.

Цель дизайна:
Сделать загрузчик не стрессовым, а приятным рабочим инструментом.
Пользователь должен понимать:
1. сколько записей проверено;
2. сколько записей корректные;
3. сколько записей с ошибками;
4. какой процент качества файла;
5. в какую "лигу качества" попал текущий файл.

Добавить игровой, но не детский элемент — персонажа:
"Муравей Сертификыч".

Стиль:
- аккуратный деловой интерфейс;
- светлый фон;
- карточки;
- цвета сайта Палаты судебных экспертов: синий, белый, светло-серый;
- игровые элементы должны быть мягкими, не мешать работе.

Главный экран:
1. Заголовок:
"Загрузчик реестра сертификатов"

2. Подзаголовок:
"Проверьте Excel-файл, обновите реестр и получите готовый JS-файл для резервной загрузки."

3. Карточка загрузки:
- поле пароля;
- выбор Excel-файла;
- кнопка "Проверить и загрузить";
- подсказка по формату файла:
  "Ожидаемые колонки: Номер документа, ФИО эксперта, Область производства судебной экспертизы, Срок действия сертификата"

После загрузки и проверки файла показать блок результата.

Расчет качества:
qualityPercent = ((totalRows - errorRows) / totalRows) * 100

Где:
- totalRows — количество непустых строк, которые участвовали в проверке;
- errorRows — количество строк, в которых найдена хотя бы одна ошибка.

Лиги качества:

1. Изумрудная лига
Условие:
qualityPercent < 90

Цвет: зеленый
Текст:
"Старт качества есть. Сертификыч уже нашёл, где укрепить реестр."

2. Сапфировая лига
Условие:
qualityPercent >= 90 и qualityPercent < 95

Цвет: синий
Текст:
"Хороший уровень. Большая часть записей прошла проверку."

3. Рубиновая лига
Условие:
qualityPercent >= 95 и qualityPercent < 98

Цвет: красный / рубиновый
Текст:
"Сильный результат. До идеального реестра осталось немного."

4. Алмазная лига
Условие:
qualityPercent >= 98 и qualityPercent < 100

Цвет: светло-голубой / алмазный
Текст:
"Почти безупречно. Реестр выглядит очень достойно."

5. Бриллиантовая лига
Условие:
qualityPercent === 100

Цвет: золотой + белый
Текст:
"Идеальный реестр. Ни одной ошибки. Сертификыч танцует."

Показать после проверки:

- Всего записей: N
- Корректных записей: N
- Записей с ошибками: N
- Качество файла: XX.XX%
- Лига качества: название лиги
- Сообщение от Сертифыча по лиге

Добавить визуальный блок персонажа:
- персонаж: маленький муравей в каске;
- можно реализовать через emoji/простую CSS-иллюстрацию, без генерации картинок и внешних файлов;
- при ошибках муравей держит лупу/папку;
- при 100% качестве — муравей делает короткую CSS-анимацию танца.

Важно:
Не использовать тяжёлые изображения и внешние библиотеки для персонажа.
Сделать через CSS/emoji, чтобы проект был лёгким.

Если есть ошибки:
Показать дружелюбный блок:

"Сертификыч нашёл записи, которые лучше поправить перед публикацией."

Ниже:
- количество строк с ошибками;
- процент ошибок;
- кнопка "Скачать Excel с ошибками";
- текст:
"Рабочий файл не обновлён. Данные на сайте остались прежними."

Если ошибок нет:
Показать блок успеха:

"Реестр успешно обновлён."

Ниже:
- количество загруженных записей;
- ссылка на файл в Supabase Storage;
- кнопка "Скачать готовый JS-файл";
- текст:
"Резервный JS-файл сохраните на случай ручного восстановления старой схемы."

Если Supabase Storage обновился успешно:
Показать:
"Файл в Storage обновлён. Тильда сможет забрать актуальные данные по новой ссылке."

Если Supabase Storage не обновился:
Показать:
"Файл в Storage не обновился, но готовый JS-файл сформирован. Его можно скачать и загрузить вручную старым способом."

Дополнительно:
- Не добавлять журнал загрузок.
- Не добавлять кабинеты и роли.
- Не добавлять графики истории.
- Не добавлять сложную аналитику.
- Не хранить статистику между загрузками.
- Все виджеты показывать только по текущему загруженному файлу.
- Сохранить скачивание готового JS-файла даже при успешной загрузке.
- Сохранить скачивание Excel-файла с ошибками при наличии ошибок.

После доработки проверить:
1. Файл с ошибками показывает лигу и виджеты.
2. Файл без ошибок показывает Бриллиантовую лигу и анимацию Сертифыча.
3. При ошибках рабочий файл в Supabase Storage не обновляется.
4. При успешной проверке файл сортируется по дате окончания сертификата от самой поздней к самой ранней.
5. Итоговый JS-файл остаётся в формате getData([...]);
