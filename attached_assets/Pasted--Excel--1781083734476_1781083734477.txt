Нужно сделать минимальный рабочий загрузчик Excel для реестра сертификатов.

ВАЖНО:
Сделай без лишних функций, без сложного дизайна, без лишних страниц.
Нужен MVP с минимальным количеством шагов и файлов.

Архитектура:
- Replit проект.
- Готовность к деплою на Vercel.
- Supabase Storage используется только для хранения готового JS-файла.
- Базу данных и таблицы Supabase НЕ создавать.
- GitHub использовать только для кода, не для хранения данных.

Задача:
Сделать одну страницу загрузки Excel-файла.

Страница:
- поле пароль администратора;
- выбор Excel-файла;
- кнопка "Загрузить";
- блок результата.

Ожидаемые колонки Excel:
1. Номер документа
2. ФИО эксперта
3. Область производства судебной экспертизы
4. Срок действия сертификата

Проверки:
1. Если в строке заполнено хотя бы одно из 4 полей, остальные поля обязательны.
2. Номер документа должен быть в формате:
   - PS 000000
   - AS 000000
   - № PS 000000
   - № AS 000000
3. ФИО эксперта должно состоять из 3 слов.
4. Срок действия сертификата должен быть в формате:
   ДД.ММ.ГГГГ-ДД.ММ.ГГГГ
5. Обе даты должны быть валидными.
6. Дата начала не может быть позже даты окончания.

Если есть ошибки:
- НЕ обновлять файл в Supabase Storage.
- Создать Excel-файл с исходными строками.
- Добавить 5-ю колонку "Ошибки".
- В каждой строке указать найденные ошибки.
- Отдать пользователю файл для скачивания.
- Показать сообщение: "Найдены ошибки. Исправьте файл и загрузите повторно."

Если ошибок нет:
1. Сформировать JS-файл строго в формате:

getData([
  {
    "Номер документа": "№ PS 003231",
    "ФИО эксперта": "Кузнецов Геннадий Витальевич",
    "Область производства судебной экспертизы": "...",
    "Срок действия сертификата": "10.01.2023-10.01.2026"
  }
]);

2. Перед обновлением сохранить текущий файл из Supabase Storage в резервную копию:
backups/table_sert_centr_sud_expert_YYYYMMDD_HHMMSS.js

3. Обновить рабочий файл в Supabase Storage:
current/table_sert_centr_sud_expert.js

4. После успешной загрузки показать:
"Файл успешно загружен. Загружено записей: N"

5. Дополнительно ОБЯЗАТЕЛЬНО отдать пользователю готовый JS-файл для скачивания:
table_sert_centr_sud_expert.js

Это нужно как резервный вариант: если что-то сломается в новой схеме, пользователь сможет вручную загрузить этот JS-файл старым способом в Битрикс/Тильду.

Переменные окружения:
SUPABASE_URL
SUPABASE_SERVICE_ROLE_KEY
SUPABASE_BUCKET
ADMIN_PASSWORD

Название bucket:
certificate-registry

Пути в Storage:
current/table_sert_centr_sud_expert.js
backups/

Важно:
- service_role key использовать только на backend.
- На frontend его не выводить.
- Рабочий файл обновлять только после успешной полной проверки Excel.
- Если загрузка в Supabase не удалась, показать ошибку и всё равно дать пользователю скачать готовый JS-файл.
- Никаких сложных кабинетов, логов, ролей, таблиц и метрик сейчас не делать.
- Сделать максимально просто и надёжно.
